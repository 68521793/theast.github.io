carla unreal4 compile and run in linux

build CARLA on Linux

First step is to install build tools, dependency and enable clang 6.0 and LLVM’s libc++ for building unreal engine 4. The following codes are used as reference, please look at the original carla document.

sudo apt-get update
sudo apt-get install wget software-properties-common
sudo add-apt-repository ppa:ubuntu-toolchain-r/test
wget -O - https://apt.llvm.org/llvm-snapshot.gpg.key|sudo apt-key add -
sudo apt-add-repository "deb http://apt.llvm.org/xenial/ llvm-toolchain-xenial-7 main"
sudo apt-get update
sudo apt-get install build-essential clang-7 lld-7 g++-7 cmake ninja-build libvulkan1 python python-pip python-dev python3-dev python3-pip libpng-dev libtiff5-dev libjpeg-dev tzdata sed curl unzip autoconf libtool rsync
//pip2 install --user setuptools
pip3 install --user setuptools

sudo update-alternatives --install /usr/bin/clang++ clang++ /usr/lib/llvm-7/bin/clang++ 170
sudo update-alternatives --install /usr/bin/clang clang /usr/lib/llvm-7/bin/clang 170

Build Unreal Engine

The next step is to build the unreal engine on local machine, get clone from UE4 github repository then go the folder and run

git clone --depth=1 -b 4.22 https://github.com/EpicGames/UnrealEngine.git ~/UnrealEngine_4.22
cd ~/UnrealEngine_4.22
./Setup.sh && ./GenerateProjectFiles.sh && make

Note: It must be the version 4.22, otherwise you will get Carla build error sometime later @make launch.

Build CARLA

to build carla we need to clone carla first, it’s preferred to use the stable brache. Optionally download aria2 will run quite faster. By calling ./Update.sh it will download the asset packages, which takes approx. 6.8 GBytes.

In order to find the Unreal Engine’s installation folder we need to set the environment variable UE4_ROOT for Carla. After setting up the enviroment we could compile the simulator and launch Unreal Engine Editor

https://github.com/carla-simulator/carla.git
sudo apt-get install aria2
cd carla
./Update.sh
export UE4_ROOT=~/UnrealEngine_4.22

It shall be noted that UE4 requires vulkan driver, which is the next generation replacement for OpenGL on linux. By missing vulkan driver you will get the failure message as below.

To install vulkan it has to be identified which GPU is installed on system. It’s best to enable a PPA for the latest Mesa drivers. please refers to vulkan.

lspci | grep VGA
sudo add-apt-repository ppa:oibaf/graphics-drivers
sudo apt update
sudo apt upgrade
sudo apt install libvulkan1 mesa-vulkan-drivers vulkan-utils
